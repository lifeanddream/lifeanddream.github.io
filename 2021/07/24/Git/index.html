Git名词概念

  • 仓库: 储存项目
  • Star: 收藏项目，方便日后查看
  • Fork: 希望对原有仓库进行修改，分支版本
  • Push Requests: 将分支版本的修改发起请求合并到原始仓库中
  • Watch: 关注某个项目，如果项目有更新，会及时通知
  • issue: 项目在使用过程中出现问题，可以提交issue, bug

本地仓库与远程仓库

在githup上创建远程仓库，获取仓库地址，使用git remote命令为该地址取别名，
fetch表明可以从远程仓库获取文件 push表明可以从本地推送到远程仓库

本地仓库推送到远程仓库

远程仓库的克隆

  1. 初始化本地仓库
  2. 将远程库内容克隆到本地
  3. 为远程仓库地址取别名

Github开发人员添加

  1. 找到对应的仓库
  2. setting -> manage access -> Invite a collaborator
  3. 输入开发人员名称
  4. 复制邀请链接，发送给开发人员

远程库修改同步

开发人员修改文件，递交远程仓库后，本地仓库同步远程仓库修改文件内容
fetch 只是将远程仓库中的内容同步到本地仓库，但工作区内容不会同步
可以使用git checkout resp_alias/branch_name切换到远程仓库查看更新, 合并之前应该先切换会分支

SSH免密操作

win10系统由于提供凭证功能，会缓存账户信息，所以只会输入一次用户名和密码

  1. 切换到用户家目录 cd ~
  2. 产生公私钥 ssh-keygen -t rsa -C “748759774@qq.com“
  3. 进入到秘钥所在的文件夹 cd ~/.ssh
  4. 打开公钥文件 vi id_rsa.pub
  5. 在github上打开个人主页, setting -> SSH and GPG keys -> New SSH Key
  6. 将公钥文件的内容复制到key中 -> Add SSH Key
  7. 为ssh地址使用命令git remote取别名
